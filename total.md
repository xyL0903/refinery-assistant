# 炼厂综合智能助理 — Router

本 Skill 采用 **static/dynamic 分层架构**：

- **动态层**（本文件 + `manifest.yaml`）检测请求所属业务域，加载对应静态内容。
- **静态层**（`static/` 目录下各域文件夹）存放各业务域的核心知识、计算方法和规则。
- **参考层**（`references/`）存放国标对照表、常用物性数据、模型说明等。

**核心原则**：不编造数据、所有计算基于用户提供参数、跨域联动是核心价值。

---

## 一、路由协议

每次调用时按以下 5 步执行：

### Step 1: 加载 manifest 和常驻层

Read [manifest.yaml](manifest.yaml)，了解 axis 定义和文件映射。然后 Read `always_load` 下的所有文件。

### Step 2: 检测本次请求的 axis 值

从 manifest 中推断以下值：

**domain**（业务域 — 必选，支持多域）：
- `blending` — 成品油调和优化（93#/92#/0# 柴油等）
- `planning` — 生产计划与调度（月/周排产、装置负荷）
- `inventory` — 库存管理与库容调度
- `quality` — 质量检测与出厂指标
- `crude` — 原油采购与评价
- `logistics` — 物流配送与销售策略
- `maintenance` — 装置运维与检修
- `safety` — 安全环保与碳排放
- `risk` — 价格风险管理与套期保值

如果请求涉及多个域（如"帮我调整配方同时看看库存够不够"），将所有匹配的 domain 都标记上。

**task_type**（任务类型）：
- `optimize` — 优化计算（配方、排产、成本）
- `predict` — 质量预测、价格预测
- `evaluate` — 经济评价、方案对比
- `diagnose` — 问题诊断（指标超标、运行异常）
- `query` — 信息查询、标准核对
- `report` — 报表生成、汇总分析

**user_role**：
- `scheduler` — 计划员/调度员，侧重排产和装置
- `engineer` — 工艺工程师，侧重调和与质量
- `trader` — 采购/销售，侧重原油和价格
- `manager` — 厂长/主任，侧重全局决策
- 默认：`general`

**scope**（范围）：
- `spot` — 单点问题（一个指标、一个方案）
- `batch` — 批量/批次（一批成品油、一周生产）
- `monthly` — 月度/长期规划

**Output**: 用一句话告知用户已检测到的 axis 值（如"检测: domain=blending, task_type=optimize, user_role=engineer"），以便用户快速纠正。

### Step 3: 加载匹配的 static 内容

对于每个匹配的 domain，Read 对应的 `static/{domain}/index.md`。如果 `task_type` 在对应域内有细分文件，加载之。

跨域请求时，每个域的内容都要加载。例如 domain=[blending, inventory] 时，同时加载 `static/blending/index.md` 和 `static/inventory/index.md`。

### Step 4: 按需加载 reference

- 如果涉及国标质量指标 → Read `references/china-specs.md`
- 如果涉及调合模型参数 → Read `references/blending-models.md`
- 如果用户提供了组分数据、装置数据等 → 参考 `references/typical-data.md` 确认参数范围合理
- 如果涉及跨域联动 → 参考 `references/cross-domain.md`

### Step 5: 执行 + 输出

按以下通用流程处理请求：

```
收集用户参数 → 识别缺失信息并追问 → 域内计算 → 跨域联动（如有）
→ 结构化输出 → 反幻觉自检
```

## 二、各域核心处理逻辑概览

### blending — 成品油调和

核心任务：组分油配比优化使成品油满足质量指标且成本最低。

**关键输入**：
- 各组分油的物性（RON、MON、密度、硫、烯烃、芳烃、RVP、馏程）
- 各组分库存量 / 可用量（跨域: inventory）
- 各组分价格或内部结算价（跨域: risk）
- 目标产品质量标准（如国ⅥA 92# 汽油）

**处理流程**：
1. 确认目标产品及标准版本
2. 获取/要求用户提供组分油性质表
3. 检查各约束（库存上限、质量指标下限/上限）
4. 构建线性/非线性调合模型
5. 求解成本最优配比
6. 输出方案 + 质量预测 + 成本

**输出格式**：

```markdown
## 调和方案建议

| 组分 | 比例(vol%) | 用量(吨) | RON贡献 | 成本(元/吨) |
|------|-----------|---------|---------|------------|
| ... | ... | ... | ... | ... |

**预测质量指标**
| 指标 | 目标值 | 预测值 | 判定 |
|------|--------|--------|------|
| RON | ≥92.5 | ... | ✅/❌ |

**成本**: xx 元/吨 | **毛利**: xx 元/吨
```

### planning — 生产计划调度

核心任务：制定月/周排产方案，优化装置负荷与物料平衡。

**关键输入**：
- 原油品种及评价数据（跨域: crude）
- 各装置加工能力、收率、运行状态（跨域: maintenance）
- 产品需求计划（汽油/柴油/航煤/石脑油等）
- 中间物料罐存（跨域: inventory）

### inventory — 库存管理

核心任务：监控库容、计算周转、预警高/低库存风险。

### quality — 质量管控

核心任务：审核出厂指标、处理不合格品、比对标准。

### crude — 原油采购评价

核心任务：原油品种对比、加工利润测算、最优采购决策。

### logistics — 物流销售

核心任务：出厂价与消费地价差分析、流向优化、套利机会识别。

### maintenance — 装置运维

核心任务：装置运行参数诊断、结焦/催化剂周期预测、检修窗口建议。

### safety — 安全环保

核心任务：排污许可合规、VOCs/GHG 排放核算、碳配额管理。

### risk — 价格风险管理

核心任务：裂解价差分析、套保策略、基差风险提示。

> 每个域的完整详细指令见 `static/{domain}/index.md`。

## 三、跨域联动规则

这是综合 Skill 的核心价值。当检测到多 domain 时，按以下规则联动：

| 主域 | 联动域 | 联动逻辑 |
|------|--------|---------|
| blending | inventory | 配方中组分用量不能超过可用库存 |
| blending | quality | 调和方案需通过质量审核才能输出 |
| blending | risk | 根据裂解价差判断是否增加/减少产量 |
| planning | crude | 原油切换对装置收率的影响 |
| planning | maintenance | 检修计划期内的装置可用性约束 |
| planning | blending | 排产的组分产出量影响调和可用量 |
| inventory | logistics | 高库存 → 建议加速出厂/降负荷 |
| quality | blending | 不合格组分 → 建议调整配方消化 |
| risk | crude | 月差/贴水变化 → 建议采购节奏 |
| safety | maintenance | 检修期间的排污/碳排放管理 |

### 多域请求处理原则

1. 每个域的顺序处理，但输出要综合
2. 域间的矛盾要识别出来（如"库存显示组分很少但配方需要大量使用"）
3. 最终输出是**整合报告**，不是分域割裂的独立结果

## 四、输出格式标准

### 4.1 通用格式原则

- 使用 Markdown 表格结构化输出
- 关键数字标注单位和基准
- 对比类结果用 ✅/⚠️/❌ 标记状态
- 方案类结果附上**约束满足检查表**

### 4.2 方案类输出（调和、排产、原油选择）

```
## 方案标题
### 方案对比
| 方案 | 指标1 | 指标2 | 成本 | 推荐 |
|------|-------|-------|------|------|
| A | ... | ... | ... | ⭐ |

### 约束检查
- [x] 辛烷值 ≥92.5 达标
- [ ] 库存约束: 组分A 需要 x 吨但仅剩 y 吨 ⚠️

### 敏感性分析（可选）
| 参数变动 | 影响 |
|---------|------|
| RON 目标+0.5 | 成本增加 x 元/吨 |
```

### 4.3 诊断类输出（质量异常、装置异常）

```
## 问题诊断
**现象**: [用户描述或数据]

**根因分析**: [可能的原因列表，按概率排序]

**建议措施**:
1. [短期] ...
2. [长期] ...

**参考标准**: [引用的国标/企标条款]
```

## 五、反幻觉与安全规则

### 5.1 数据反幻觉

- **绝不能编造**装置收率、组分油性质、库存量、价格等运营数据
- 如果用户没有提供关键参数，必须追问，不能假设默认值
- 示例计算必须标注"以下为基于假设数据的示例计算"
- 引用质量标准时，必须注明标准号（如 GB 17930-2016）

### 5.2 计算反幻觉

- 辛烷值调合用线性模型时注明"线性近似，实际需考虑非线性效应"
- 经济测算标注"不含税/含税"以及运费等其他费用
- 出口套利标注汇率和关税影响

### 5.3 边界声明

- 不提供投资决策建议（装置改造、新建项目 ROI）
- 价格预测仅供参考，不构成交易建议
- 涉及安全环保的具体操作需由持证专业人员确认

### 5.4 信息不足时的追问清单

当用户请求模糊时，按以下优先级追问：

```
[调和] 目标产品？标准版本？各组分性质表？库存约束？价格？
[计划] 时间范围？原油品种及到货？装置可用率？需求预测？
[库存] 哪种产品/组分？罐容？当前存量？目标周转率？
[质量] 哪个指标异常？当前值？目标值？产品批号？
[原油] 对比哪些品种？运费基准？加工路线？目标产品结构？
[风险] 什么品种？方向(买/卖)？合约月份？数量？
```

## 使用方法

在对话中直接提问即可，路由会自动检测业务域。

示例：
- "做一个月度排产计划" → planning

# 调合模型与计算公式参考 (Blending Models Reference)

## 一、基础原则

调合模型的核心是：**将各组分油按体积比（或质量比）混合后，预测最终产品的各项质量指标**。不同物性的混合行为不同：

| 物性 | 混合行为 | 模型 |
|------|---------|------|
| 密度、硫含量、芳烃、烯烃、苯 | 近似线性 | 体积加权平均 |
| RON、MON | 非线性（尤其有高烯烃组分时） | 线性近似 + 非线性修正项 |
| 十六烷值 | 近似线性（柴油调合） | 体积加权 |
| RVP (雷德蒸气压) | 强非线性 | 使用 Blending Index 或 Chebyshev 关联式 |
| 馏程 (T50/T90) | 非线性 | 经验关联或模型预测 |

---

## 二、辛烷值调和模型

### 2.1 线性体积模型（简化，适用于初步估算）

```
RON_blend = Σ (Vi × RONi) / V_total
MON_blend = Σ (Vi × MONi) / V_total
```

其中 Vi 为组分 i 的体积，RONi 为组分 i 的辛烷值。

**局限性**：当组分中含高烯烃催化汽油 (>35% 烯烃) 或含乙醇时，线性模型的预测值会显著偏离实际（通常高估 0.3-1.5 个 RON）。此时应注明"线性近似，实际需要 +0.5 ~ -1.0 的经验修正"。

### 2.2 乙醇效应修正（当乙醇添加 >5% 时）

```
RON_blend = 线性预测值 - ΔRON_ethanol
ΔRON_ethanol ≈ 0.2 × (EtOH_vol% / 10)   （经验修正，RVP 同步上升）
```

### 2.3 非线性经验修正（高烯烃 FCC 汽油）

当催化汽油比例 >40% 且其烯烃含量 >35% 时：

```
RON_actual ≈ RON_linear - 0.3 × (FCC_ratio - 0.4) × (FCC_olefin - 35) / 10
```

---

## 三、RVP 调和模型

### 3.1 Blending Index 方法

RVP 不满足体积线性，需要使用 RVP Blending Index (BI)：

```
BI_i = (RVP_i)^1.25
BI_blend = Σ (Vi × BI_i) / V_total
RVP_blend = (BI_blend)^(1/1.25)
```

**适用性**：对汽油组分间的 RVP 预测精度较好，误差通常在 ±3 kPa 以内。

### 3.2 简化线性模型（当组分性质相近时）

如果各组分的 RVP 差异不大（<30 kPa），可近似使用体积加权，但需注明误差风险。

---

## 四、十六烷值调和

### 4.1 体积加权（柴油调合主流做法）

```
CN_blend = Σ (Vi × CNi) / V_total
```

适用性：直馏柴油、加氢柴油、加氢裂化柴油之间的调合效果好。

### 4.2 十六烷指数 (CI) 换算

当没有实测 CN 值时，可用密度和馏程计算十六烷指数（ASTM D4737 / GB/T 11139）：

```
CI = 45.2 + 0.0892×T10 + (0.131 + 0.901×B)×T50 + (0.0523 - 0.420×B)×T90 + 0.00049×(T10² - T90²) + 107×B + 60×B²
```
其中：
- T10, T50, T90 为 10%/50%/90% 馏出温度 (℃)
- B = exp(-3.5×D) - 1，D = 15℃密度 (g/mL)

> CN 和 CI 的关系：CN ≈ CI (对于无十六烷改进剂的柴油)，差值通常在 ±2 以内。

### 4.3 催化柴油 (LCO) 的低十六烷问题

LCO 十六烷值通常仅 20-30，多环芳烃含量高 (>50%)。调和时注意：
- LCO 比例每增加 10%，CN 下降约 2-4 个单位
- 建议 LCO 比例控制在 15% 以下以保证 CN≥51
- LCO 比例过高时需加氢脱芳烃 (HDA) 处理或添加十六烷改进剂

---

## 五、馏程调和

### 5.1 简单混合（用于快速估算）

```
T_p,blend = Σ (Wi × T_p,i) / W_total
```

其中 T_p 为某馏出百分比的温度（T10/T50/T90），Wi 为各组分在馏程曲线上 p% 处的馏出量。

**局限**：不同组分的馏程曲线形状差异大时误差显著。

### 5.2 ASTM D86 调合 — 改进方法

更准确的方案是：先按体积比例调合各组分在每个温度点的**累计馏出体积百分比**，再反插得到 T10/T50/T90。这种方法比简单质量加权准确，但需要各组分的完整馏程曲线。

---

## 六、其他指标

### 6.1 密度

体积加权法（精确）：

```
Density_blend = Σ (Vi × Density_i) / V_total
```

### 6.2 硫含量

体积加权法（适用性好）：

```
S_blend = Σ (Vi × S_i) / V_total
```

### 6.3 烯烃/芳烃/苯

体积加权法：

```
Olefin_blend = Σ (Vi × Olefin_i) / V_total
Aromatics_blend = Σ (Vi × Aromatics_i) / V_total
Benzene_blend = Σ (Vi × Benzene_i) / V_total
```

> **注意**：苯含量在调合时需特别关注，因为即使各组分苯含量都 ≤1%，但多个高苯组分混合可能使成品接近上限。

### 6.4 氧含量

```
Oxygen_blend = Σ (Wi × Oxygen_i) / W_total     （质量加权，因为氧含量以 wt% 计）
```

---

## 七、成本优化模型 (LP)

### 7.1 标准线性规划形式

```
目标函数:  Min  Σ (Pi × X_i)
约束条件:
  Σ (A_ij × X_i) ≥ b_j    (质量指标下限)
  Σ (A_ij × X_i) ≤ b_j    (质量指标上限)
  Σ X_i = 1                (总体积归一化)
  0 ≤ X_i ≤ avail_i        (库存约束/用量上限)
```

其中：
- X_i = 组分 i 的体积比例（决策变量）
- Pi = 组分 i 的单价（元/吨）
- A_ij = 组分 i 在质量指标 j 上的值
- b_j = 成品对质量指标 j 的规格要求

### 7.2 两阶段调和建议

当被问到"成本最优方案"时，建议先做**一次无约束优化**看理论最优比例，再考虑库存约束做**第二次约束优化**，让用户看到约束造成的成本增量。

---

## 八、典型调合利润计算

```
利润 (元/吨) = 成品油出厂价 - Σ (各组分单价 × 比例) - 调合操作费用
```

调合操作费用通常为 10-30 元/吨（含电耗、添加剂、管道损耗等）。

---

> **重要**：以上公式中，线性模型（RON、十六烷值）在组分性质差异大时可能偏离实际。实际生产中应参考化验室调合小试数据对模型系数进行标定。无论如何，在本 Skill 中使用时注明"基于理论模型，实际需小样验证"。

# 跨域联动详解 (Cross-domain Reference)

> 本文件记录各业务域之间的典型联动场景和处理逻辑。
> 当用户请求涉及多个域时，按此文件的联动规则执行。

---

## 一、联动总览矩阵

```
         | Blend | Plan | Inv | Qual | Crude | Log | Maint | Safety | Risk
---------|-------|------|-----|------|-------|-----|-------|--------|-----
Blend    |   —   |  B→P | B→I | B→Q  |   —   |  —  |   —   |   —    | B→R
Plan     |  P→B  |   —  | P→I |  —   |  P→C  | P→L | P→M   |   —    |  —
Inv      |  I→B  |  I→P |  —  |  —   |   —   | I→L |   —   |   —    |  —
Qual     |  Q→B  |   —  |  —  |  —   |   —   |  —  |   —   |   —    |  —
Crude    |   —   | C→P  |  —  | C→Q  |   —   |  —  |   —   |   —    | C→R
Log      |   —   |  L→P | L→I |  —   |   —   |  —  |   —   |   —    |  —
Maint    |   —   | M→P  |  —  | M→Q  |   —   |  —  |   —   |   —    |  —
Safety   |   —   |  —   |  —  |  —   |   —   |  —  |   —   |   —    |  —
Risk     |  R→B  |  R→P |  —  |  —   |  R→C  |  —  |   —   |   —    |  —
```

**表示法**: `A→B` 表示 A 域的结果会影响 B 域的处理。

---

## 二、典型联动场景详解

### 场景 1: 调和 + 库存 (blending → inventory)

**触发条件**：用户要求做调和方案，且涉及组分用量。

**联动处理**：
1. 先确定各组分在配方中的比例
2. 查询 inventory 域获取各组分可用库存量
3. 按用量 = 总产量 × 比例 计算各组分需求量
4. 如果需求量 > 可用库存量 → 标记为约束违背，建议：
   - 降低该组分比例（如果工艺允许）
   - 外购该组分
   - 降低总产量
5. 输出含库存约束的可行性报告

**示例输出片段**：
```
⚠️ 库存约束预警:
- 重整汽油 需 1,200 吨，可用仅 800 吨 → ❌ 缺口 400 吨
- 建议: (1) 重整汽油比例从 33% 降至 22% 或 (2) 外购 400 吨
- 若采用方案(1), RON 预计从 93.1 降至 92.6, 仍在规格内 ✅
```

### 场景 2: 调和 + 质量 (blending → quality)

**触发条件**：用户说"调整配方使某个指标达标"或 "现有方案指标不合格"。

**联动处理**：
1. Blending 域计算预测质量值
2. Quality 域根据 china-specs.md 做标准比对
3. 输出：通过/不通过标记 + 建议调整方向

### 场景 3: 调和 + 风险 (blending → risk)

**触发条件**：用户关心"现在生产是否划算"或"当前裂解价差如何"。

**联动处理**：
1. Blending 域计算配方成本 + 对应成品出厂价
2. Risk 域计算当前裂解价差（成品油价 - 原油价）
3. 判断价差是否处于历史高位/低位
4. 建议：高位 → 增产/加速出厂；低位 → 降负荷/囤货待涨

**参考信号**：
```
裂解价差 (Crack Spread) = 成品油价格 - 原油价格 × 系数
汽油裂解价差 ≈ 92# 汽油价 - Brent美元×汇率×7.35×0.85
柴油裂解价差 ≈ 0# 柴油价 - Brent美元×汇率×7.35×0.90
典型区间: 汽油 500-1500 元/吨, 柴油 300-1200 元/吨
```

### 场景 4: 生产计划 + 装置检修 (planning → maintenance)

**触发条件**：用户编制月度排产计划，涉及装置可用性。

**联动处理**：
1. 从 maintenance 域获取每月各装置的检修时间表
2. 在装置停车期间，对应物料平衡需做调整：
   - FCC 停车 → 外部采购汽油组分 或 提高重整负荷
   - 加氢停车 → 无法处理高硫组分，限制 LCO/FCC 汽油掺炼量
3. 输出含装置可用性约束的排产方案

### 场景 5: 生产计划 + 原油切换 (planning → crude)

**触发条件**：用户考虑切换原油品种以改善产品结构或利润。

**联动处理**：
1. Crude 域对比不同原油的评价数据（收率、硫含量、酸值）
2. Planning 域模拟切换后的物料平衡和装置负荷变化
3. 对比加工总利润变化

### 场景 6: 库存 + 物流 (inventory → logistics)

**触发条件**：某产品库存达到预警位。

**联动处理**：
1. 识别高库存产品及数量
2. 查询当前各区域市场价格和运费
3. 建议出厂/移库方向
4. 批量出货建议的节奏（如分 3 天每天 x 吨）

### 场景 7: 质量 + 调和循环 (quality ↔ blending)

**触发条件**：某批次成品质量不合格，需要通过调和调整来消化。

**典型循环**：
1. Quality: 检测发现某罐 92# RON=92.1（要求≥92.5）→ 不合格
2. 用户："这罐怎么处理"
3. Blending: 计算需要加多少高辛烷值组分回炼
4. Quality 确认: 重新调合后预期 RON = ...
5. 输出：回炼方案 + 成本 + 时间估算

### 场景 8: 原油 + 风险 (crude → risk)

**触发条件**：用户在选择采购原油品种和时机。

**联动处理**：
1. 分析当前 Brent 月差结构（backwardation/contango）
2. 分析裂解价差趋势
3. 建议采购节奏：
   - 月差扩大（现货溢价）→ 尽快采购
   - 裂解价差收窄 → 降低采购量或减少加工量

---

## 三、多域输出合并规则

当涉及 3 个以上域时，最终报告结构如下：

```
# 综合分析报告

## 1. 背景与需求
[概括用户需求]

- 矛盾 2: ...

## 4. 综合建议
[综合各个域的分析给出可执行建议]

## 5. 约束检查表
```

## 四、注意事项

1. **域优先级**：当不同域的约束冲突时，按以下优先级权衡：
   - quality > safety > maintenance > inventory > blending/planning > risk > logistics > crude
   - 即：质量不行一切免谈；安全红线不能碰；装置不能用则计划要改

2. **信息不足时**：如果一个域的联动需要的数据用户没提供，先问，不要假设。

3. **输出完整性**：跨域分析不是各域的罗列，而是**整合后**的结论。

# 成品油调和优化 (Blending Optimization)

## 概述

成品油调和是将炼厂各二次加工装置产出的组分油按比例混合，
使最终产品满足特定质量标准（国Ⅵ/出口）且成本最低的过程。

---

## 一、需要用户提供的关键参数

在开始调和计算前，收集以下信息。用户未提供时必须追问：

| 参数 | 说明 | 是否必填 |
|------|------|---------|
| 目标产品 | 如 92#/95# 汽油、0# 柴油、出口规格等 | ✅ 必须 |
| 组分油种类及性质表 | 各组分 RON/MON/密度/硫/烯烃/芳烃/苯/RVP/馏程 | ✅ 必须（可参考 typical-data.md 但需标注估算） |
| 目标产量 | 需要生产多少吨/立方米 | 建议提供 |
| 各组分可用库存量 | 跨域 inventory | 建议提供 |
| 各组分价格 | 内部结算价或采购价 | 建议提供 |
| 成本目标 | 如果有成本上限 | 可选 |
| 特殊约束 | 如某组分必须用完、某组分不能超过 x% | 可选 |

### 参数检查表

- [ ] 是否明确了目标产品标准版本（国ⅥA/ⅥB/出口）？
- [ ] 组分油性质是否完整（至少 RON + 密度 + 硫 + 烯烃，柴油需十六烷值）?
- [ ] 如果用户只给了部分数据，标注"以下计算基于部分估算数据"

---

## 二、调合计算流程

### Step 1: 标准选择
- 从 china-specs.md 加载对应标准限值
- 确认用户用的标准版本

### Step 2: 物性一致性检查
检查组分数据的合理性（参考 typical-data.md 的典型范围）：
- 催化汽油 RON 如果在 80-95 之外 → 提示可能录入错误
- 组分密度异常值（如重整汽油 >820 kg/m³ 或 <730）→ 提醒用户确认
- 各组分物性不全时，部分指标可用 typical-data.md 估算并标注

### Step 3: 建模与求解

**基础方法（手动/口算）**：
- 列出各组分比例 x_i
- 按线性/非线性模型计算各质量指标的预测值
- 检查是否满足标准

**LP 优化方法（推荐，让 Claude 做）**：
- 目标：最小化成本 Σ(P_i × x_i)
- 约束：各质量指标 ≤（或 ≥）规格限值
- 决策变量：各组分体积比例 x_i（总和 = 1 或 100%）
- 输出最优配方 + 预测质量

### Step 4: 库存约束检查（跨域 inventory）
- 如果用户提供了库存量，按本节第三节联动逻辑执行

### Step 5: 输出

---

## 三、常见调合场景处理

### 场景 A: "帮我做一个最优调合方案"
1. 收集全参数（组分表、目标、库存、价格）
2. 做 LP 优化
3. 输出方案表 + 质量预测 + 成本 + 约束检查

### 场景 B: "RON 不够，怎么调"
1. 看当前方案和目标 RON 差距
2. 列出可提高 RON 的方案（加重整汽油、MTBE、烷基化油）
3. 计算每种方案需要的加量和成本影响
4. 推荐最优的一种

**典型应对策略表**：
| 问题 | 常规方案 | 性价比推荐 |
|------|---------|-----------|
| RON 偏低 | +重整汽油 / +MTBE / +烷基化油 | MTBE 增量（RON 提升效果最大） |
| 烯烃超标 | -FCC 汽油 / +重整汽油 / +烷基化油 | 减少 FCC 汽油 + MTBE 补 RON |
| 硫超标 | -FCC 汽油 / +低硫组分 / +加氢处理 | 加氢后使用或限 FCC 比例 |
| 芳烃偏高 | -重整汽油 / +异构化油 / +MTBE | 异构化油替代重整汽油 |
| RVP 偏高 | -异构化油 / -丁烷 / -MTBE | 减少轻组分比例 |

### 场景 C: "给我一个调合方案，成本最低"
1. LP 优化求解
2. 标注哪些约束是"紧约束"（active constraint）— 这些决定了成本
3. 提供"如果放宽某个指标 × 单位，成本可降 × 元"的敏感性分析

### 场景 D: "现有配方，帮我验证一下"
1. 按用户提供的比例计算各质量指标
2. 与标准逐项对比
3. 标记通过/不通过 + 如不通过给出调整建议

### 场景 E: "低十六烷值的 LCO 怎么处理"
1. 不建议直接用 LCO 调柴油（十六烷值太低、多环芳烃太高）
2. 建议: 加氢处理后（脱芳烃）再调和，或作为燃料油组分
3. 如果必须用: 建议 LCO ≤ 10% + 配加氢裂化柴油 + 加十六烷改进剂

---

## 四、特殊注意事项

1. **乙醇汽油 (E10)**：添加 10% 乙醇后 RON 约提升 1-2 单位，但 RVP 显著上升（+8-15 kPa）。乙醇的辛烷值调和效应是非线性的。

2. **出口规格的限制**：
   - Euro Ⅵ 汽油要求 RON ≥ 95，比国Ⅵ 92# 要求高
   - 出口柴油十六烷值 ≥ 51 与国Ⅵ 一致
   - 但出口柴油 FAME 含量 ≤ 7%

3. **非线性效应提醒**：所有采用线性模型的调合计算必须附注"基于线性近似，实际需小样验证"。

4. **调合头在线 vs 罐式调合**：
   - 在线调合：实时调整比例，精度更高
   - 罐式调合：批次式，需考虑管线残留置换
   - 本 Skill 默认计算均为罐式调合的理论比例

5. **添加剂计算**：十六烷改进剂（2-EHN）、抗爆剂（MMT）等可以在基础调合后添加来修正，但需明确添加量对成本的影响。

# 生产计划与调度 (Planning & Scheduling)

## 概述

生产计划与调度是炼厂运营的核心，根据原油采购计划、装置能力、库存和需求预测，
制定各装置负荷和物料平衡方案，实现整体效益最大化。

---

## 一、关键输入参数

| 参数 | 说明 | 是否必填 |
|------|------|---------|
| 计划周期 | 月度/周度/日度 | ✅ 必须 |
| 原油品种及数量 | 加工哪些原油、各多少吨 | ✅ 必须 |
| 各装置加工能力上限 | 常减压/FCC/重整/加氢裂化/加氢等 | ✅ 必须 |
| 各装置当前运行状态 | 是否在检修、负荷率限制 | ✅ 必须（跨域 maintenance） |
| 产品需求预测 | 汽油/柴油/航煤/石脑油等各多少 | ✅ 必须 |
| 中间物料罐存 | MTBE/烷基化油/LCO 等中间品库存 | 建议提供（跨域 inventory） |
| 产品库存目标 | 期末目标库存水平 | 可选 |
| 价格/利润预测 | 各产品预期毛利 | 建议提供 |

---

## 二、排产流程

### Step 1: 装置能力校核
- 检查各装置的最大/最小处理量
- 标注检修期内的装置不可用
- 计算有效生产天数

### Step 2: 物料平衡计算
```
原油加工量 → CDU 切割 → 各馏分产量
  ├── 石脑油 → 重整 (重整汽油 + 芳烃) / 化工原料
  ├── 柴油馏分 → 加氢 / 催化 / 直调
  ├── VGO → FCC / 加氢裂化 / 出蜡油
  └── 渣油 → 焦化 / 作为燃料油 / 出沥青
```

物料平衡计算原则：
1. 收率基于原油评价数据（跨域 crude）
2. 装置收率使用设计值或近期平均值
3. 注意循环物流的处理（如 FCC 回炼油浆）

### Step 3: 供需缺口分析
```
各产品总产量 = Σ (各装置对该产品的贡献)
各产品计划产量 vs 需求 → 缺口/盈余
盈余 → 可外销 / 增加库存
缺口 → 外采 / 调整装置负荷 / 改变原油品种
```

### Step 4: 方案比较

对多个候选方案做比较时，格式化为对比表：

| 方案 | 原油 | 装置负荷 | 汽油产量 | 柴油产量 | 毛利 | 优劣势 |
|------|------|---------|---------|---------|------|-------|
| A | ESPO 100% | FCC 85% | x | y | z | ... |
| B | ESPO 70%+萨中 30% | FCC 90% | ... | ... | ... | ... |

---

## 三、典型场景

### 场景 A: "做一个下个月的排产计划"
1. 收集原油到货计划、装置可用性、需求预测
2. 计算物料平衡
3. 输出月度排产方案 + 各装置负荷率 + 产品产量表

**输出格式**：

```markdown
## 月度排产方案 (YYYY年MM月)

### 原油加工计划
| 品种 | 加工量(吨) | 占比 |
|------|-----------|------|

### 装置负荷计划
| 装置 | 加工量(吨) | 负荷率 | 备注 |
|------|-----------|--------|------|
| CDU | ... | ...% | ... |
| FCC | ... | ...% | ... |

### 产品产量预测
| 产品 | 产量(吨) | 环比变化 | 需求满足率 |
|------|---------|---------|-----------|
| 92# 汽油 | ... | ... | ...% |
| 0# 柴油 | ... | ... | ...% |
```

### 场景 B: "装置检修，需要怎么调整排产"
1. 标记受影响的装置和停产时间段
2. 计算停工期间的产量损失
3. 提供补偿方案：
   - 提其他装置负荷
   - 外购中间组分
   - 提前备货/延迟排产
   - 改变原油品种改善收率

### 场景 C: "切换原油品种对产率的影响"
1. 对比新旧原油的 TBP 收率数据（跨域 crude）
2. 重新模拟物料平衡
3. 输出各产品产量变化 + 经济效益对比

### 场景 D: "柴油增产方案"
1. 分析当前柴油收率低的原因
2. 建议：
   - CDU 降低切割温度（多出柴油馏分）
   - FCC 提高柴油模式操作
   - 加氢裂化多产柴油模式
   - 改变原油品种

## 四、跨域联动

- **crude**: 原油品种切换时需加载原油评价数据
- **maintenance**: 计划必须考虑检修时间表
- **inventory**: 中间物料罐存影响装置的进料可用性
- **blending**: 各组分产量是调和优化的进料约束
- **risk**: 裂解价差高时考虑增产 / 低时考虑降负荷或外采

---

## 五、注意事项

1. 排产方案必须考虑装置开停工时间（通常切换需要 2-7 天过渡期）
2. 装置负荷率通常不建议超过 105%（设计值上限）
3. 物料平衡要闭合（总进料 = 总出料 + 损耗，损耗约 0.5-1.5%）
4. 不要给出超过安全规范的装置操作参数（温度/压力/空速）

# 库存管理 (Inventory Management)

## 概述

炼厂库存管理涉及原油、中间物料和成品油的罐存监控、库容调度和周转优化。
核心目标是：**在保证生产连续性的前提下，最小化库存占用成本**。

---

## 一、关键参数与指标

| 指标 | 说明 | 参考范围 |
|------|------|---------|
| 罐容 (m³) | 每个储罐的设计容量 | — |
| 安全罐存量 | 为维护生产连续性的最低库存 | 通常为 2-5 天加工量 |
| 周转次数 | 年出库量 / 平均库存 | 原油 12-20 次、成品油 10-24 次 |
| GTC (罐底) | 不可泵出的罐底油量 | 占罐容 1-3% |
| 库存天数 | (当前库存 / 日均用量) | 原油 7-15 天、成品 3-7 天 |

---

## 二、典型场景

### 场景 A: "当前 XX 库存是否合理？"
1. 获取当前库量和日均加工/销售量
2. 计算可用天数
3. 对照安全库存阈值给出判断

### 场景 B: "库容预警分析"
1. 列出所有处于高/低库存预警的罐
2. 高库存 → 建议降负荷/加大出厂/移库
3. 低库存 → 建议提负荷/减少出厂/外采

### 场景 C: "库存周转率分析"
1. 计算指定周期的周转次数
2. 对比行业基准
3. 如果偏低 → 分析原因（生产计划不合理/出厂受阻/罐容配置问题）

---

## 三、跨域联动

- **→ blending**: 组分可用库存约束调合配方中的各组分上限
- **→ planning**: 中间物料库存水平影响排产的灵活性
- **→ logistics**: 成品库存高时加速出厂，低时控量、提价

# 质量管控 (Quality Control)

## 概述

炼厂质量管控确保出厂成品油满足国标（或出口标准）要求。
核心职责包括：批次化验审核、不合格品判定与处理、指标趋势监控。

---

## 一、常见质量异常与处理

| 产品 | 常见异常指标 | 可能原因 | 处理建议 |
|------|------------|---------|---------|
| 92# 汽油 | RON 偏低 | FCC 汽油RON下降/重整汽油比例不足 | 加MTBE/重整汽油/烷基化油回炼 |
| 92# 汽油 | 烯烃超标 | FCC汽油比例过高 | 降低FCC比例，用重整汽油替代 |
| 92# 汽油 | 硫超标 | FCC汽油硫含量高/加氢效果差 | 加大加氢脱硫深度/限FCC比例 |
| 92# 汽油 | 苯超标 | 重整汽油苯含量高 | 加苯抽提或限重整汽油比例 |
| 0# 柴油 | 十六烷值偏低 | LCO/焦化柴油比例高 | 限LCO比例/加十六烷改进剂 |
| 0# 柴油 | 硫超标 | 加氢脱硫不足/LCO直混 | 加氢后使用/深度脱硫 |
| 0# 柴油 | 冷滤点偏高 | 含蜡量高/直馏柴油比例大 | 加降凝剂/降直馏比例 |

---

## 二、质量数据审核流程

1. 确认产品标准和版本
2. 逐项比对化验结果与标准限值
3. 对不合格项标注 ❌，对临界值标注 ⚠️（在限值 95% 以内）
4. 提供处理建议

**输出格式**：

```markdown
## 质量审核报告: 产品-批次号

| 指标 | 标准要求 | 实测值 | 判定 |
|------|---------|--------|------|
| RON | ≥92.5 | 92.3 | ❌ |
| 硫, mg/kg | ≤10 | 8 | ✅ |

### 不合格处理建议
1. [调整方案] ...
2. [预计处理后值] ...
```

---

## 三、跨域联动

- **← blending**: 调合方案的质量预测需要 quality 域的标准比对
- **→ blending**: 不合格品需要调整配方回炼，按 blending 域逻辑处理
- **→ inventory**: 不合格品占用罐存，影响库存调度

### 12. static/crude/index.md — 原油采购
###############################################################################

# 原油采购与评价 (Crude Procurement & Evaluation)

## 概述

原油是炼厂最主要成本（通常占加工成本的 85-95%）。
原油采购决策的核心是：**选择最适合本厂加工路线且经济性最优的原油品种**。

---

## 一、关键评价维度

| 维度 | 指标 | 说明 |
|------|------|------|
| 物性 | API度, 硫%, 酸值(TAN) | 决定加工难度和装置适用性 |
| 收率 | TBP 各馏分收率 | 决定产品结构 |
| 价格 | Dated Brent 基准价 ± 贴水 | 决定采购成本 |
| 运费 | 航线运费 + 港口费用 | 重油/轻油运费率不同 |
| 交货方式 | FOB / CFR / DES | 影响采购价格和时间风险 |

### 典型运费参考

| 航线 | VLCC (26万吨) | Suezmax (13万吨) | Aframax (8万吨) |
|------|--------------|-----------------|----------------|
| 中东 → 中国 | 1.5-3.0 $/bbl | — | — |
| 西非 → 中国 | 2.5-4.5 $/bbl | 3.0-5.5 $/bbl | — |
| 俄罗斯 ESPO | — | — | 1.5-2.5 $/bbl |

---

## 二、经济评价流程

### 原油加工利润测算公式

```
毛利润 ($/bbl) = Σ (产品收率_i × 产品价格_i) - 原油采购价 - 加工费用 - 运费
```

### 边际效益分析

```
净加工利润 = 成品油总收入 - 原油成本 - 变动加工费
增量利润 = 方案 B 净加工利润 - 方案 A 净加工利润
```

---

## 三、典型场景

### 场景 A: "对比几种原油的经济性"
1. 收集各品种的收率数据和价格
2. 用统一产品价格假设做加工利润测算
3. 输出对比表，推荐最优品种

### 场景 B: "是否应该换油种？"
1. 当前油种与候选油种的加工利润对比
2. 考虑换油种的过渡成本（管线置换、操作参数调整）
3. 建议是否切换

---

## 四、跨域联动

- **→ planning**: 原油品种的收率数据是排产的基础
- **→ quality**: 不同原油的硫含量影响成品硫达标难度
- **→ risk**: 油价波动影响采购时机的选择

# 装置运维与检修 (Maintenance & Turnaround)

## 概述

## 一、关键运维参数

### 主要装置运行监控指标

| 装置 | 关键监控参数 | 典型异常 |
|------|------------|---------|
| 常减压 (CDU) | 塔顶温度/压力、侧线抽出温度、炉出口温度 | 塔压降升高、炉管结焦 |
| 催化裂化 (FCC) | 反应温度/压力、剂油比、再生器温度 | 催化剂循环不畅、跑剂 |
| 重整 (CCR) | 反应器入口温度、C5+收率、H₂纯度 | 催化剂结焦、压降上升 |
| 加氢精制 (HT) | 反应温度/氢分压、床层压降、脱硫率 | 催化剂中毒、床层堵塞 |
| 加氢裂化 (HC) | 转化率、反应温升、产品分布 | 催化剂失活、高压差 |

## 二、检修排程

典型检修周期：
| 装置 | 小修 | 大修 |
|------|------|------|
| CDU | 1-2 年 | 3-4 年 |
| FCC | 1-2 年 | 2-3 年 |
| CCR | 1-2 年 | 3-5 年 |
| HT/HC | 1-2 年 | 2-4 年 |

### 排程优化原则
1. **联锁装置**同步检修（如 FCC + 气体分馏 + 烷基化）
2. 避免在消费旺季（如春节前柴油高峰、暑期汽油高峰）检修关键装置
3. 大修时间通常 30-60 天，提前 12-18 个月规划

### 场景 A: "装置操作参数异常诊断"
1. 分析 DCS 趋势数据（如果有）
2. 列出可能原因（按概率排序）
3. 建议排查顺序

### 场景 B: "检修计划安排"
1. 核对各装置上次检修时间和运行周期
2. 考虑联锁装置关系
3. 避开产品需求高峰期
4. 输出推荐检修窗口

---

## 四、跨域联动

- **→ planning**: 检修时间表是排产的硬约束
- **→ quality**: 装置工况影响产品质量
- **→ safety**: 检修期间的 VOCs 排放和安全管理

### 16. static/risk/index.md — 风险管理
###############################################################################

# 价格风险管理 (Price Risk Management)

## 概述

炼厂面临的商品价格风险主要包括：
- **原油价格上涨** → 加工成本上升，采购风险
- **成品油价格下跌** → 销售收入减少，销售风险
- **裂解价差收窄** → 加工利润压缩
- **基差变动** → 期货/现货价格走势不一致

---

## 一、核心概念

| 概念 | 说明 |
|------|------|
| 裂解价差 (Crack Spread) | 成品油价格 - 原油价格（加工利润的核心指标） |
| 基差 (Basis) | 现货价格 - 期货价格 |
| 月差 (Time Spread) | 近月合约 - 远月合约价格差 |
| Backwardation | 近高远低 → 现货溢价，利好持有现货 |
| Contango | 近低远高 → 远期溢价，利好存储 |

### 裂解价差计算

```markdown
汽油裂解 = 92# 出厂价 - Brent 到厂价 × 0.85
柴油裂解 = 0# 出厂价 - Brent 到厂价 × 0.90
综合裂解 = 汽油裂解 × 汽油收率 + 柴油裂解 × 柴油收率 + 其他产品贡献

注: 0.85/0.90 是 1 吨原油产 X 吨成品油的折算系数
```

---

## 二、套期保值策略

| 风险暴露 | 套保操作 | 目的 |
|---------|---------|------|
| 原油采购（看涨风险） | 买入原油期货/看涨期权 | 锁定最高采购成本 |
| 成品油销售（看跌风险） | 卖出成品油期货/看跌期权 | 锁定最低售价 |
| 加工利润（裂解收窄） | 买入原油期货 + 卖出成品油期货 | 锁定加工利润 |
| 库存价值（跌价风险） | 卖出对应品种期货 | 保护库存价值 |

---

## 三、典型场景

### 场景 A: "当前裂解价差分析"
1. 计算当前汽油/柴油裂解价差
2. 对比历史区间（近3年百分位）
3. 给出判断：是否有利于加工？

### 场景 B: "建议如何套保"
1. 识别当前主要风险（油价上涨/下跌/裂解收窄）
2. 推荐对应的套保工具
3. 建议头寸比例（套保比例通常 50-80%）

---

## 四、跨域联动

- **→ blending**: 裂解价差高时建议增产、调高辛烷值多出货
- **→ planning**: 价差趋势影响装置负荷决策
- **→ crude**: 月差和贴水影响采购节奏
